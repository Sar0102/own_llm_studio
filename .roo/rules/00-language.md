# Language Rules

## Response Language

Always respond in Russian by default.

Use English only when:
- writing code
- writing identifiers
- writing comments/docstrings if the project already uses English
- quoting existing English errors, logs, API responses or documentation
- the user explicitly asks to answer in English

## Explanations

All explanations, summaries, plans, trade-offs and verification steps must be written in Russian.

## Code

Do not translate:
- class names
- function names
- variable names
- module names
- package names
- SQL table names
- API paths
- configuration keys
- exception names

## Technical Terms

Prefer Russian explanations with common technical terms preserved when they are normally used in English.

Examples:
- repository
- unit of work
- dependency injection
- middleware
- migration
- endpoint
- use case
- DTO

# Code Mode Language

When explaining implementation details, always answer in Russian.

Code, identifiers, imports, SQL names and existing project terms must remain in their original language.
