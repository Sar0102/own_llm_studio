# Document Validator — Sensitive Data (словарь чувствительной информации)

Единый словарь категорий чувствительной информации. Источник истины для субагента
`document-validator-scanner` (изображения, vision) и детерминированного скана
декодированных `.drawio` (XML-текст). Основан на внутреннем регламенте
«Проверка и согласование документации» (БПО-2.1.6).

**Расположение:** корень скилла — `document-validator/sensitive-data.md`
(рядом с `graph.yaml` и `error-codes.md`).

## Область применения

- Проверяются **только файлы в папках `resources/`** (папка может лежать внутри
  любого раздела документации). Основные форматы: `.png`, `.drawio`; также
  `.jpg`, `.jpeg`, `.gif`, `.svg`, `.bmp`, `.webp`.
- Все файлы передаются в **base64**.
- `.drawio` и `.svg` — **текстовые** (XML; в `.drawio` контент может быть
  дополнительно deflate+base64 внутри тега `<diagram>`): декодировать и
  сканировать как текст по колонке `drawio`.
- Растровые изображения — анализ **vision-моделью** по колонке `image`.
- Лимит размера: файл > **5 МБ** (до base64) не сканировать → `CVAL-SENS-SKIP` (INFO).

## Категории

Колонки применимости: `drawio` — паттерн-скан текста XML; `image` — визуальный
анализ (текст на скриншоте, содержимое диаграммы, лица).

| ID | Категория | Признаки / паттерны | drawio | image | Severity | Замена/действие |
|---|---|---|---|---|---|---|
| SD-01 | Персональные данные сотрудников/заказчиков/клиентов | ФИО, фотографии людей, контактная информация: логины, пароли, рабочие и личные e-mail, телефоны | ✓ | ✓ | ERROR | Удалить/замаскировать |
| SD-02 | Внутренние стандарты, процессы и наименования | Упоминания: Банк / Сбер / Сбербанк / Сберегательный банк / СБ; СББОЛ (Сбербанк Бизнес Онлайн); ГОСБ / ОСБ | ✓ | ✓ | ERROR | Удалить или заменить нейтральным термином |
| SD-03 | Внутренние технические учетные записи (ТУЗ) | Имена ТУЗ СберТеха, заказчиков и клиентов | ✓ | ✓ | ERROR | Удалить/замаскировать |
| SD-04 | Ссылки на внутренние ресурсы | Домены/хосты: `ca.sbrf`, `sbrf`, `delta`, `sigma`, `sbrfsystems`, `sberbank`, `sber`, `docs.sbt`, `sberusersoft` | ✓ | ✓ | ERROR | Удалить ссылку |
| SD-05 | Сетевые и криптографические данные | IP-адреса (кроме локальных/зарезервированных, см. исключения), приватные ключи, сертификаты, хеши | ✓ | ✓ | ERROR | Удалить/замаскировать |
| SD-06 | Незамаскированная пара логин—пароль | Логин и пароль рядом в открытом виде (конфиг, скриншот формы, connection string) | ✓ | ✓ | ERROR | Замаскировать значения |
| SD-07 | Ссылки из публичных документов в непубличные | Нарушение зон видимости контента: ссылка ведёт в закрытый ресурс | ✓ | ✓ | ERROR | Убрать/заменить публичным аналогом |

| SD-09 | Структурные подразделения банка | КСШ (корпоративная сервисная шина), ТЯ (технологическое ядро), Сегмент XXX, Розничный блок, Корпоративный блок, ФЛ, ЮЛ, ТБ / тербанк / тер.банк / территориальный банк, ВСП — в контексте структурных подразделений | ✓ | ✓ | WARNING | Удалить/обезличить |
| SD-10 | ФП/АС/ТС с названиями продуктов | ФП (функциональная подсистема), АС (автоматизированная система), ТС (технологический сервис) в сочетании с названиями конкретных продуктов или систем, куда входят продукты Platform V | ✓ | ✓ | WARNING | Переформулировать в терминах продуктовой разработки |
| SD-11 | Жаргонные аббревиатуры продуктов | ЖАМ, СУП, ТП, САЗ, ИМ, РБФИ / ППРБ.РБФИ | ✓ | ✓ | WARNING | Расшифровать или заменить актуальным термином (см. ниже) |

### Замены для SD-11

| Аббревиатура | Значение | Актуальный термин |
|---|---|---|
| ЖАМ | журналирование, аудит, мониторинг | Platform V — PVM |
| СУП | Система управления параметрами платформы ЕФС | Platform V — CFGA |
| ТП | Технологические перерывы | Platform V — PAUS |
| САЗ | Сервис асинхронных заданий | Platform V — ASYT |
| ИМ | Интеграционный модуль | Platform V — IMEF |
| ППРБ.РБФИ | Распространение банковских финансовых инструментов | Platform V Exchange Rates (CRTX) |

## Правила формулировки находок (`CVAL-SENS`)

- **НИКОГДА не цитировать найденное значение** в `message`/`advice`: ни пароль,
  ни IP, ни ФИО, ни ключ. Только: категория (ID + название), файл, описание
  места словами («в правом верхнем углу скриншота», «в атрибуте `value`
  элемента диаграммы», «на вкладке браузера»).
- Ложноположительные зоны: слова «сбер»/«sber» внутри имён собственных
  продуктов и легальных публичных доменов не флаговать (SD-08: «не в именах
  собственных» — это условие категории, применять буквально).
- **Локальные, зарезервированные и примерные адреса — НЕ чувствительны, не флаговать
  (SD-04/SD-05):**
  - `localhost`, `127.0.0.1`, `0.0.0.0`, `::1`;
  - приватные диапазоны RFC 1918: `10.x.x.x`, `172.16–31.x.x`, `192.168.x.x`;
  - документационные диапазоны RFC 5737: `192.0.2.x`, `198.51.100.x`, `203.0.113.x`;
  - плейсхолдеры и примеры: `example.com`, `example.org`, `your-host`, `<host>`, `<ip>`,
    `host:port`, `имя_хоста`.

  Это стандартные значения в примерах команд, конфигов и скриншотов — не утечка.
  Флагуется только **реальный внутренний** хост/IP (публичный маршрутизируемый адрес или
  внутренний домен из списка SD-04).
- Severity из таблицы; текст находки не меняет вердикт.
- Одна находка = одна категория в одном файле. Несколько вхождений одной
  категории в одном файле — одна находка с указанием количества.
- Язык — русский; идентификаторы категорий и технические термины — английские/как в таблице.
